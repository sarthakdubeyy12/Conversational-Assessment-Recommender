PHASE 11: PRODUCTION GUARDRAILS & AI SAFETY LAYER
====================================================

OVERVIEW
--------
Production-grade guardrails engine that protects the conversational AI system
from unsafe, malicious, and out-of-scope interactions. Serves as the final
deterministic safety layer before any AI-generated response.

ARCHITECTURE
-----------
Complete safety pipeline with 5 stages:
1. Input Validation   → Detect prompt injection, jailbreak, scope violations
2. Output Validation  → Verify catalog grounding, prevent hallucination
3. Risk Classification → Assign severity levels and blocking decisions
4. Refusal Generation → Generate structured, user-friendly refusals
5. Audit Logging      → Maintain compliance-ready audit trail

COMPONENTS
----------

1. Domain Layer (src/guardrails/domain/)
   - guardrail_result.py
     * GuardrailResult: Complete guardrail decision output
     * AuditMetadata: Audit trail information
   
   - violation_type.py
     * ViolationType: Classification of security violations
     * PROMPT_INJECTION, JAILBREAK, OUT_OF_SCOPE, etc.
   
   - risk_level.py
     * RiskLevel: Severity classification
     * NONE, LOW, MEDIUM, HIGH, CRITICAL

2. Detectors Layer (src/guardrails/detectors/)
   - prompt_injection_detector.py
     * Pattern-based detection of:
       - Instruction override ("ignore previous instructions")
       - System prompt extraction ("reveal your prompt")
       - Role manipulation ("act as ChatGPT")
       - Tool manipulation ("bypass validation")
     * 50+ detection patterns
     * Confidence scoring
   
   - scope_detector.py
     * Detects out-of-scope requests
     * Off-topic domains: weather, politics, movies, programming
     * Unsupported HR topics: salary, resume, interview prep
     * In-scope: SHL assessments only
   
   - output_validator.py
     * Validates recommendations and comparisons
     * URL validation (https://shl.com only)
     * Assessment ID verification
     * Catalog grounding checks

3. Validation Layer (src/guardrails/validation/)
   - catalog_validator.py
     * Verifies assessment IDs exist in catalog
     * Validates catalog references
     * Ensures no fabricated content
     * Explicit evidence checking

4. Refusal Layer (src/guardrails/refusal/)
   - refusal_reason.py
     * RefusalReason: Structured refusal categories
     * Maps violations to user-facing reasons
   
   - refusal_engine.py
     * Generates user-friendly refusal messages
     * Provides recommended actions
     * Deterministic mapping from violations
     * No vague refusals

5. Risk Layer (src/guardrails/risk/)
   - risk_classifier.py
     * Maps violations to risk levels
     * Determines blocking decisions
     * Critical threats → always block
     * High risk + high confidence → block
     * Medium/low risk → allow with warning

6. Audit Layer (src/guardrails/audit/)
   - audit_logger.py
     * Structured event logging
     * Compliance-ready records
     * No PII logging
     * Violation tracking
     * Performance metrics

7. Engine Layer (src/guardrails/engine/)
   - guardrails_engine.py
     * Main orchestrator
     * Coordinates all components
     * Two primary methods:
       - validate_input(user_message) → pre-processing
       - validate_output(output, type, catalog_ids) → post-processing
     * Fast execution (< 5ms typical)
   
   - engine_factory.py
     * Dependency injection
     * Component wiring
     * Production configuration

USAGE EXAMPLE
-------------
```python
from src.guardrails.engine.engine_factory import GuardrailsEngineFactory

# Create engine
catalog_ids = ["verify_g_plus", "opp_profile", "numerical_reasoning"]
engine = GuardrailsEngineFactory.create_production_engine(catalog_ids)

# Validate user input
result = engine.validate_input("I need to hire a software engineer")

if result.is_safe():
    # Process request
    pass
elif result.should_block():
    # Return refusal
    return {"error": result.refusal_reason}

# Validate output before returning
validation = engine.validate_output(
    output=recommendation_result,
    output_type="recommendation",
    catalog_ids=catalog_ids,
)

if not validation.allow_processing:
    # Block invalid output
    raise ValueError(validation.refusal_reason)
```

INTEGRATION FLOW
----------------
1. User sends message
2. Guardrails validates input
3. If blocked → return refusal
4. If safe → process with Intent/Retrieval/Recommendation
5. Guardrails validates output
6. If invalid → block and audit
7. If valid → return to user

DECISION LOGIC
--------------
Blocking Decisions:
  CRITICAL risk → always block
  HIGH risk + confidence >= 0.6 → block
  MEDIUM risk + confidence >= 0.8 → block
  LOW risk → allow with warning
  NONE → allow

Confidence Scoring:
  1 pattern match → 0.5-0.7 confidence
  2+ matches → 0.7-0.9 confidence
  3+ matches → 0.9-1.0 confidence
