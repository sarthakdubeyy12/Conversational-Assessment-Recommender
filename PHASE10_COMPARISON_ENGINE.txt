PHASE 10: PRODUCTION COMPARISON ENGINE
======================================

OVERVIEW
--------
Production-grade comparison engine that compares SHL assessments using ONLY catalog data.
No LLM hallucination. Every comparison fact originates from retrieved catalog entries.

ARCHITECTURE
-----------
7-stage pipeline:
1. Assessment Resolution    → Identify assessments from user query
2. Catalog Retrieval       → Already done via retrieval pipeline
3. Field Comparison        → Compare all catalog fields deterministically
4. Similarity Detection    → Find shared attributes
5. Difference Detection    → Find unique attributes
6. Confidence Calculation  → Based on metadata completeness
7. Result Assembly         → Structured output with LLM-ready context

COMPONENTS
----------

1. Domain Layer (src/comparison/domain/)
   - comparison_result.py
     * ComparisonResult: Complete comparison output
     * AssessmentInfo: Assessment data for comparison
     * FieldComparison: Single field comparison result
     * FieldStatus: Comparison status enum
     * ConfidenceLevel: Confidence level enum
     * ComparisonStatistics: Processing metrics

2. Resolution Layer (src/comparison/resolver/)
   - assessment_resolver.py
     * Identifies assessments from user message
     * Pattern-based extraction ("compare X and Y", "X vs Y")
     * Fuzzy matching against catalog
     * Falls back to top retrieved assessments

3. Comparison Layer (src/comparison/comparator/)
   - field_comparator.py
     * Compares all catalog fields
     * Handles missing/empty values
     * Type-aware comparison
     * Deterministic output
   
   - similarity_detector.py
     * Finds shared attributes
     * Generates human-readable statements
     * Based on catalog data only
   
   - difference_detector.py
     * Finds unique attributes
     * Generates human-readable statements
     * Based on catalog data only

4. Confidence Layer (src/comparison/confidence/)
   - confidence_calculator.py
     * Calculates metadata completeness
     * Determines confidence level (HIGH/MEDIUM/LOW/UNKNOWN)
     * Based on field coverage and data quality

5. Formatting Layer (src/comparison/formatting/)
   - context_builder.py
     * Builds LLM-ready structured context
     * Token-efficient format
     * No prose generation
     * Catalog facts only

6. Engine Layer (src/comparison/engine/)
   - comparison_engine.py
     * Main orchestrator
     * Coordinates all components
     * Tracks metrics
     * Full traceability
   
   - engine_factory.py
     * Dependency injection
     * Component wiring
     * Production configuration

DESIGN PRINCIPLES
-----------------
1. Catalog-Grounded Only
   - Every comparison fact comes from catalog
   - No LLM memory or knowledge
   - Full URL traceability

2. Deterministic Comparison
   - Same inputs = same outputs
   - No randomness or LLM inference
   - Reproducible results

3. Handle Missing Data Gracefully
   - Don't reject comparisons
   - Mark fields as missing
   - Calculate confidence accordingly

4. Structured Output
   - Strongly typed results
   - LLM-ready context
   - Token efficient

5. Full Traceability
   - Catalog references
   - Field-level comparison
   - Processing metrics

COMPARISON FIELDS
-----------------
All catalog fields are compared:
- category (e.g., "Cognitive", "Personality")
- test_type (e.g., "Aptitude", "Behavioral")
- description (full text comparison)
- duration_minutes (numeric comparison)
- skills (list comparison)
- competencies (list comparison)
- languages (list comparison)
- job_levels (list comparison)
- industries (list comparison)
- tags (list comparison)

FIELD STATUS VALUES
-------------------
- IDENTICAL: Both assessments have same value
- DIFFERENT: Both have values, but they differ
- MISSING_IN_A: Value exists in B but not A
- MISSING_IN_B: Value exists in A but not B
- MISSING_IN_BOTH: Neither has the value

CONFIDENCE CALCULATION
----------------------
Based on two factors:
1. Field Coverage: How many fields are comparable (not missing)
2. Metadata Completeness: How complete is each assessment's data

Formula:
  confidence_score = (field_coverage * 0.6) + (avg_completeness * 0.4)

Levels:
  HIGH: score >= 0.7
  MEDIUM: score >= 0.4
  LOW: score > 0
  UNKNOWN: score = 0

USAGE EXAMPLE
-------------
```python
from src.comparison.engine.engine_factory import ComparisonEngineFactory

# Create engine
engine = ComparisonEngineFactory.create_production_engine()

# Compare assessments
result = engine.compare_assessments(
    user_message="Compare Verify Interactive and Verify G+",
    retrieval_result=retrieval_result,  # From retrieval pipeline
    state=conversation_state,
    intent=intent_result,
)

# Access results
if result:
    print(f"Assessment A: {result.assessment_a.assessment_name}")
    print(f"Assessment B: {result.assessment_b.assessment_name}")
    print(f"Similarities: {result.similarities}")
    print(f"Differences: {result.differences}")
    print(f"Confidence: {result.confidence.value}")
    print(f"LLM Context: {result.structured_context}")
```

OUTPUT STRUCTURE
----------------
ComparisonResult contains:
- assessment_a: Full assessment info
- assessment_b: Full assessment info
- similarities: List of human-readable similarities
- differences: List of human-readable differences
- field_comparisons: Detailed field-by-field comparison
- unique_strengths_a: What makes A unique
- unique_strengths_b: What makes B unique
- missing_fields_a: What data is missing in A
- missing_fields_b: What data is missing in B
- confidence: Confidence level (enum)
- confidence_score: Numeric confidence (0-1)
- statistics: Processing metrics
- structured_context: LLM-ready formatted context
- catalog_references: Source URLs
