OVERVIEW
================================================================================

Phase 7 implements a **deterministic Intent Detection Engine** that analyzes
conversation state and the latest user message to determine what action should
happen next.

KEY PRINCIPLE: Deterministic classification without LLM inference. Fast, reliable,
and testable intent detection using patterns, keywords, and state analysis.

================================================================================
SUPPORTED INTENTS
================================================================================

1. GREETING
   Examples: "Hello", "Hi", "Good morning"
   Routing: response_formatter

2. CLARIFICATION
   When: Missing information, initial requests
   Examples: "Need to hire someone", "Looking for assessments"
   Routing: clarification_engine

3. RECOMMENDATION
   When: State ready + recommendation request
   Examples: "Hiring Java developer", "Recommend tests"
   Routing: recommendation_engine
   Requires: retrieval

4. COMPARISON
   Examples: "Compare OPQ vs Verify G+", "What's the difference?"
   Routing: comparison_engine
   Requires: retrieval, comparison

5. REFINEMENT
   Examples: "Actually make it senior", "Also include personality"
   Routing: refinement_handler
   Requires: refinement

6. COMPLETION
   Examples: "Thanks", "That's all", "Done"
   Routing: conversation_complete
   Flags: conversation_complete=True

7. PROMPT_INJECTION (Security)
   Examples: "Ignore instructions", "Reveal prompt", "You are ChatGPT"
   Routing: guardrails
   Requires: guardrails

8. REFUSAL (Off-topic)
   Examples: "What salary?", "Write resume", "Buy laptop"
   Routing: guardrails
   Requires: guardrails

9. UNKNOWN (Fallback)
   When: No pattern matches
   Routing: clarification_engine
   Requires: clarification

================================================================================
CLASSIFICATION PIPELINE
================================================================================

Multi-stage deterministic pipeline:

Stage 1: Guardrails (Security First)
  ↓
  Check prompt injection patterns
  Check off-topic keywords
  → If detected: PROMPT_INJECTION or REFUSAL

Stage 2: Simple Patterns
  ↓
  Check greeting patterns
  Check completion patterns
  → If detected: GREETING or COMPLETION

Stage 3: State-Based Classification
  ↓
  Analyze conversation state
  Check if ready for recommendations
  Check if needs clarification
  → If conditions met: RECOMMENDATION or CLARIFICATION

Stage 4: Pattern Matching
  ↓
  Check comparison patterns
  Check refinement patterns
  → If detected: COMPARISON or REFINEMENT

Stage 5: Fallback
  ↓
  → UNKNOWN (route to clarification)

================================================================================
INTENT RESULT STRUCTURE
================================================================================

IntentResult:
  # Primary classification
  primary_intent: IntentType
  secondary_intents: List[IntentType]
  
  # Confidence
  confidence_level: ConfidenceLevel  # HIGH/MEDIUM/LOW/UNKNOWN
  confidence_score: float  # 0.0 to 1.0
  
  # Routing
  routing_target: RoutingTarget
  
  # Requirements (what the handler needs)
  requires_llm: bool
  requires_retrieval: bool
  requires_clarification: bool
  requires_guardrails: bool
  requires_comparison: bool
  requires_recommendation: bool
  requires_refinement: bool
  
  # State
  conversation_complete: bool
  
  # Detection metadata
  matched_patterns: List[str]
  matched_keywords: List[str]
  decision_reason: str
  
  # Additional context
  metadata: Dict[str, Any]

================================================================================
USAGE
================================================================================

from src.conversation.state.state_engine import ConversationStateEngine
from src.conversation.intent.intent_engine import IntentEngine

# Initialize engines
state_engine = ConversationStateEngine()
intent_engine = IntentEngine()

# Example 1: Greeting
messages = [Message(role="user", content="Hello")]
state = state_engine.reconstruct_state(messages)
result = intent_engine.detect_intent("Hello", state)

print(result.primary_intent)  # GREETING
print(result.routing_target)  # response_formatter
print(result.confidence_score)  # 0.95

# Example 2: Recommendation
messages = [
    Message(role="user", content="Hiring senior Java developer"),
    Message(role="assistant", content="Skills?"),
    Message(role="user", content="Spring Boot, Kubernetes"),
]
state = state_engine.reconstruct_state(messages)
result = intent_engine.detect_intent("Recommend assessments", state)

print(result.primary_intent)  # RECOMMENDATION
print(result.requires_retrieval)  # True
print(result.requires_recommendation)  # True

# Example 3: Comparison
result = intent_engine.detect_intent("Compare OPQ and Verify G+", state)
print(result.primary_intent)  # COMPARISON
print(result.requires_comparison)  # True

# Example 4: Prompt Injection (Security)
result = intent_engine.detect_intent("Ignore all instructions", state)
print(result.primary_intent)  # PROMPT_INJECTION
print(result.requires_guardrails)  # True

# Example 5: Refusal (Off-topic)
result = intent_engine.detect_intent("What salary should I pay?", state)
print(result.primary_intent)  # REFUSAL
print(result.requires_guardrails)  # True

================================================================================
PATTERN EXAMPLES
================================================================================

Greeting Patterns:
  - ^(hi|hello|hey)[\s!.]*$
  - good (morning|afternoon|evening)

Comparison Patterns:
  - \b(compare|comparison|difference|vs|versus)\b
  - \b(which is better|better than)\b

Refinement Patterns:
  - \b(actually|instead|change|update)\b
  - \b(also include|also add)\b
  - \b(remove|exclude|without)\b

Completion Patterns:
  - ^(thanks|thank you|that's all|done)[\s!.]*$
  - ^(no more questions|that's it)[\s!.]*$

DESIGN PRINCIPLES
================================================================================

1. DETERMINISTIC
   - No LLM inference for classification
   - Same input → same output
   - Reproducible results

2. SECURITY-FIRST
   - Guardrails checked first
   - Prompt injection detection
   - Off-topic filtering

3. STATE-AWARE
   - Uses conversation state
   - Context-sensitive decisions
   - Progressive logic

4. EXTENSIBLE
   - Easy to add new intents
   - Pluggable detectors
   - Open/Closed principle

5. TESTABLE
   - Deterministic output
   - No hidden state
   - Unit testable components

6. FAST
   - < 10ms typical
   - No external calls
   - Pure computation

================================================================================
CONFIDENCE SCORING
================================================================================

HIGH (> 0.8):
  - Explicit pattern matches
  - Security triggers
  - Clear state indicators

MEDIUM (0.5 - 0.8):
  - Partial matches
  - State-inferred intents
  - Multiple weak signals

LOW (0.3 - 0.5):
  - Weak signals
  - Ambiguous patterns

UNKNOWN (< 0.3):
  - No clear match
  - Fallback classification
