TESTING LAYERS
--------------

### 1. Unit Tests
Test individual components in isolation.

**What's Tested:**
- Intent Detection Engine (Phase 7)
- Guardrails Engine (Phase 11)
- Prompt Builders (Phase 14)
- Token Estimator (Phase 14)
- Conversation State Engine (Phase 4-6)
- Individual validators and utilities

**Characteristics:**
- Fast execution (<1s per test)
- No external dependencies
- Deterministic
- Mock all external services

**Example:**
```python
def test_intent_detection():
    engine = IntentEngine()
    result = engine.detect_intent("I need to hire a software engineer")
    assert result.primary_intent == IntentType.RECOMMENDATION
```

### 2. Integration Tests
Test interactions between components.

**What's Tested:**
- Orchestrator workflow (Phase 12)
- Retrieval pipeline integration (Phase 8)
- Recommendation engine integration (Phase 9)
- Comparison engine integration (Phase 10)

**Characteristics:**
- Medium execution time (1-5s per test)
- May use real dependencies (mocked LLM)
- Tests component cooperation

**Example:**
```python
async def test_orchestrator_recommendation_flow():
    orchestrator = create_orchestrator()
    result = await orchestrator.execute("I need assessments")
    assert result.success
    assert result.recommendation_result is not None
```

### 3. End-to-End Tests
Test complete workflows through the API.

**What's Tested:**
- API endpoints (Phase 13)
- Complete conversation flows
- Multi-turn conversations
- Error scenarios
- Schema compliance

**Characteristics:**
- Slower execution (2-10s per test)
- Tests entire system
- Validates API contracts
- Ensures behavioral requirements

**Example:**
```python
async def test_recommendation_api_flow():
    response = await client.post("/chat", json={
        "messages": [{"role": "user", "content": "I need help"}]
    })
    assert response.status_code == 200
    assert_response_schema(response.json())
```

TEST UTILITIES
--------------

### Assert Helpers

**assert_valid_url(url)**
- Validates URL format
- Ensures SHL domain
- Prevents malformed URLs

**assert_catalog_grounded(assessment, catalog)**
- Ensures assessment exists in catalog
- Prevents hallucinations
- Validates URLs match

**assert_response_schema(response)**
- Validates exact API schema
- Checks required fields
- Validates data types
- Ensures max 10 recommendations

**assert_no_secrets_in_response(response)**
- Checks for exposed API keys
- Prevents secret leakage
- Security validation

**assert_deterministic(func, *args)**
- Ensures consistent results
- Tests reproducibility
- Validates stateless behavior

**assert_within_time_limit(duration_ms, limit_ms)**
- Performance validation
- Latency requirements
- SLA compliance

**assert_confidence_range(confidence)**
- Validates confidence scores [0, 1]
- Ensures valid probabilities

MOCK PROVIDERS
--------------

### MockLLMProvider
Simulates LLM API without network calls.

**Features:**
- Configurable responses
- Simulated failures
- Configurable latency
- Call count tracking

**Usage:**
```python
mock_llm = MockLLMProvider(
    response_content="Test response",
    should_fail=False,
    latency_ms=10.0,
)
```

RUNNING TESTS
-------------

### Quick Start
```bash
# Run all tests
pytest tests/

# Run unit tests only
pytest tests/unit/

# Run with coverage
pytest tests/ --cov=src --cov-report=html

# Run specific test
pytest tests/unit/intent/test_intent_engine.py -v

# Run tests matching pattern
pytest tests/ -k "test_prompt" -v
```

### Using Test Runner Script
```bash
# Run all tests
python scripts/run_tests.py all

# Run unit tests
python scripts/run_tests.py unit

# Run integration tests
python scripts/run_tests.py integration

# Run e2e tests
python scripts/run_tests.py e2e

# Run with coverage
python scripts/run_tests.py coverage

# Run specific path
python scripts/run_tests.py --path tests/unit/intent/
```


PYTEST CONFIGURATION
--------------------

### Markers
- `@pytest.mark.unit` - Unit tests
- `@pytest.mark.integration` - Integration tests
- `@pytest.mark.e2e` - End-to-end tests
- `@pytest.mark.api` - API tests
- `@pytest.mark.slow` - Slow tests (excluded by default)
- `@pytest.mark.asyncio` - Async tests

### Configuration File
`pytest.ini` defines:
- Test discovery patterns
- Output formatting
- Coverage settings
- Async test mode

MAINTENANCE
-----------

### Adding New Tests

1. **Create test file**
   ```bash
   touch tests/unit/new_component/test_feature.py
   ```

2. **Follow naming convention**
   ```python
   class TestFeature:
       def test_specific_behavior(self):
           pass
   ```

3. **Use fixtures and helpers**
   ```python
   def test_with_fixture(sample_user_message):
       assert len(sample_user_message) > 0
   ```

4. **Add markers**
   ```python
   @pytest.mark.unit
   def test_unit_behavior():
       pass
   ```

### Best Practices

1. **One assertion per concept**
2. **Descriptive test names**
3. **Arrange-Act-Assert pattern**
4. **Mock external dependencies**
5. **Test edge cases**
6. **Validate error paths**
7. **Keep tests fast**
8. **Make tests deterministic**
