===========================================================
IMPLEMENTED COMPONENTS
===========================================================

1. PIPELINE RESULT (pipeline_result.py)
   - RankedDocument entity
   - PipelineStatistics entity
   - PipelineResult entity (complete output)
   - Rich diagnostics
   - Strongly typed
   - Immutable design

2. QUERY BUILDER (query/query_builder.py)
   - Extracts information from ConversationState
   - Generates multiple optimized queries
   - Primary: role + seniority + skills
   - Secondary: assessment types
   - Tertiary: skills + competencies
   - Text normalization
   - Noise removal

3. METADATA FILTER BUILDER (filters/metadata_filter_builder.py)
   - Builds ChromaDB-compatible filters
   - Assessment type filtering
   - Language filtering
   - Category filtering (cognitive/personality/coding)
   - Job level filtering
   - Seniority mapping
   - State-driven filter construction

4. HYBRID RANKER (ranking/hybrid_ranker.py)
   - Combines semantic similarity + metadata + skills
   - Configurable weights:
     * Semantic: 0.5
     * Metadata: 0.3
     * Skills: 0.2
   - Metadata scoring: assessment type, language, job level, industry
   - Skill overlap scoring
   - Normalized ranking scores (0-1)
   - Explainable ranking factors
   - Sorted output

5. DUPLICATE REMOVER (compression/duplicate_remover.py)
   - Deduplicates by assessment_id
   - Keeps highest-ranked chunk per assessment
   - Preserves ranking order
   - Tracks deduplication metrics

6. CONTEXT COMPRESSOR (compression/context_compressor.py)
   - Compresses retrieval context for LLM
   - Limits to top N assessments (default: 5)
   - Formats assessment information
   - Metadata prioritization
   - Token estimation (4 chars ≈ 1 token)
   - Optimized for downstream consumption

7. RETRIEVAL VALIDATOR (validation/retrieval_validator.py)
   - Validates retrieval quality
   - Checks for empty results
   - Validates similarity scores
   - Checks metadata completeness
   - Checks URL validity
   - Generates non-blocking warnings
   - Diagnostic information

8. RETRIEVAL PIPELINE (retrieval_pipeline.py)
   - Complete orchestration
   - 7-stage pipeline:
     1. Query Building
     2. Semantic Retrieval
     3. Metadata Filtering
     4. Hybrid Ranking
     5. Duplicate Removal
     6. Context Compression
     7. Validation
   - Performance tracking per stage
   - Rich statistics
   - Production-ready output

9. PIPELINE FACTORY (pipeline_factory.py)
   - Dependency injection
   - Configuration management
   - Single point of creation
   - Production instance factory

===========================================================
PIPELINE FLOW
===========================================================

Input:
  ConversationState (hiring requirements)
  IntentResult (user intent)
  
↓

Stage 1: QUERY BUILDING (1-3ms)
  - Extract key information from state
  - Generate optimized queries
  - Normalize text
  
↓

Stage 2: SEMANTIC RETRIEVAL (50-100ms)
  - Execute queries against ChromaDB
  - Apply metadata filters
  - Retrieve top-K chunks
  
↓

Stage 3: HYBRID RANKING (5-10ms)
  - Calculate semantic scores
  - Calculate metadata relevance
  - Calculate skill overlap
  - Weighted combination
  - Sort by final score
  
↓

Stage 4: DUPLICATE REMOVAL (1-2ms)
  - Group by assessment_id
  - Keep highest-ranked chunk
  
↓

Stage 5: CONTEXT COMPRESSION (2-5ms)
  - Limit to top N assessments
  - Format for LLM consumption
  - Estimate token count
  
↓

Stage 6: VALIDATION (1ms)
  - Check result quality
  - Generate warnings
  
↓

Output:
  PipelineResult (complete structured output)

# Create pipeline

    vector_store=vector_store,
    top_k=20,
    max_assessments=5,
)

# Execute
result = await pipeline.execute(state, intent)

# Use results
for doc in result.get_top_assessments(5):
    print(doc.result.assessment_name)
    print(doc.ranking_score)
