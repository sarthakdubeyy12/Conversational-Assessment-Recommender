===========================================================
IMPLEMENTED COMPONENTS
===========================================================

1. DOMAIN ENTITIES (domain/recommendation_result.py)

   AssessmentRecommendation:
   - Complete recommendation information
   - Traceable to catalog & retrieval
   - Contains: ID, name, URL, type, category
   - Scores: ranking, similarity, metadata, skill
   - Explanation & matching factors
   - Catalog metadata preserved

   RecommendationStatistics:
   - Processing metrics
   - Quality metrics
   - Filtering metrics

   RecommendationResult:
   - Complete engine output
   - List of recommendations
   - Confidence level (high/medium/low)
   - Statistics & validation
   - Decision rationale

2. CANDIDATE SELECTOR (selection/candidate_selector.py)

   Responsibilities:
   - Filter low-quality results
   - Remove incomplete entries
   - Apply quality thresholds
   - Validate URLs (must be shl.com)

   Filters:
   - Minimum similarity threshold
   - URL requirement (https://shl.com)
   - Category requirement (optional)
   - Name existence

3. RECOMMENDATION RANKER (ranking/recommendation_ranker.py)

   Responsibilities:
   - Apply final ranking adjustments
   - Boost relevant categories
   - Apply diversity weighting
   - Enforce ranking limits (max 10)

   Ranking Logic:
   - Base score: Retrieval ranking score
   - Category boost: +0.1 for matching category
   - Diversity boost: +0.05 for first of each category
   - Sort by final score
   - Limit to max recommendations

4. RECOMMENDATION VALIDATOR (validation/recommendation_validator.py)

   Validates:
   - URL correctness (must be https://shl.com)
   - Metadata completeness
   - Duplicate detection
   - Minimum/maximum count (1-10)

   Generates non-blocking warnings for:
   - Invalid URLs
   - Missing metadata
   - Duplicates

5. EXPLANATION BUILDER (explanation/explanation_builder.py)

   Generates:
   - Recommendation reasons
   - Matching factors
   - Human-readable explanations
   - Based ONLY on catalog metadata

   Explanation Sources:
   - Skill matches
   - Competency matches
   - Category matches
   - Job level matches

   Example reasons:
   - "Measures Java, Problem-solving; Assesses Technical thinking"
   - "Cognitive reasoning assessment; Suitable for senior roles"

6. RECOMMENDATION ENGINE (engine/recommendation_engine.py)

   5-Stage Pipeline:
   
   Stage 1: Candidate Selection
   - Filter low-quality candidates
   - Validate URLs
   - Check metadata completeness
   
   Stage 2: Ranking
   - Apply final ranking
   - Category boosts
   - Diversity weighting
   
   Stage 3: Explanation Generation
   - Build recommendation reasons
   - Extract matching factors
   - Generate human-readable text
   
   Stage 4: Validation
   - Validate recommendations
   - Check URLs
   - Generate warnings
   
   Stage 5: Result Assembly
   - Package recommendations
   - Calculate statistics
   - Determine confidence
   - Build rationale

7. ENGINE FACTORY (engine/engine_factory.py)

   Dependency Injection:
   - Wires up all components
   - Configures thresholds
   - Single point of creation

===========================================================
PIPELINE FLOW
===========================================================

Input:
  PipelineResult (from Retrieval Pipeline)
  ConversationState (hiring context)
  IntentResult (user intent)

↓ Candidate Selection
  Filter by quality, URL, metadata

↓ Ranking
  Apply final ranking + boosts

↓ Explanation Generation
  Build reasons from metadata

↓ Validation
  Check URLs, metadata, duplicates

↓ Result Assembly
  Package complete result

Output:
  RecommendationResult
    - List of recommendations
    - Confidence level
    - Statistics
    - Validation status
    - Rationale
