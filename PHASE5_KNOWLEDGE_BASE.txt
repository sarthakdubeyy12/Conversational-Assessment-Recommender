OVERVIEW
================================================================================

Phase 5 transforms the SHL assessment catalog into a production-grade semantic
knowledge base optimized for Retrieval-Augmented Generation (RAG).

This is the foundation that powers all future recommendation, comparison, and
clarification features.

================================================================================
ARCHITECTURE
================================================================================

Clean Architecture with clear separation of concerns:

src/knowledge_base/
├── domain/
│   ├── document.py          # Document and DocumentChunk entities
│   └── interfaces.py        # Domain interfaces (abstraction layer)
├── embeddings/
│   ├── provider.py          # SentenceTransformer implementation
│   ├── batch_processor.py  # Efficient batch processing
│   └── embedding_cache.py   # Persistent embedding cache
├── vector_store/
│   ├── chroma_client.py     # ChromaDB implementation
│   └── collection_manager.py # Collection lifecycle management
├── document_builder.py      # Assessment → Document transformation
├── semantic_chunker.py      # Document → Chunks (semantic splitting)
├── index_builder.py         # Orchestrates full indexing pipeline
└── semantic_search.py       # Semantic search over knowledge base

================================================================================
KEY COMPONENTS
================================================================================

1. DOCUMENT BUILDER
   - Transforms assessments into searchable documents
   - Preserves ALL metadata for filtering
   - Generates rich content for embedding
   - One document per assessment

2. SEMANTIC CHUNKER
   - Creates focused semantic chunks
   - Three chunk types: overview, skills, full
   - Preserves context and metadata in every chunk
   - Optimized for catalog data structure

3. EMBEDDING PROVIDER
   - Sentence Transformers implementation
   - Lazy model loading
   - Batch processing support
   - Device selection (CPU/GPU)
   - Provider abstraction for future models

4. EMBEDDING CACHE
   - Persistent disk-based cache
   - Avoids recomputing unchanged embeddings
   - Model-specific cache isolation
   - Speeds up incremental updates

5. BATCH PROCESSOR
   - Efficient batch embedding generation
   - Configurable batch sizes
   - Retry logic on failures
   - Progress reporting

6. CHROMADB CLIENT
   - Production ChromaDB integration
   - Persistent storage
   - Rich metadata storage
   - Similarity search with filtering
   - Batch operations

7. COLLECTION MANAGER
   - Collection lifecycle management
   - Statistics and health checks
   - Rebuild support
   - Validation

8. INDEX BUILDER
   - Orchestrates complete pipeline
   - Catalog → Documents → Chunks → Embeddings → Vector Store
   - Incremental update support
   - Comprehensive logging and progress reporting

9. SEMANTIC SEARCH
   - Query embedding generation
   - Vector similarity search
   - Metadata filtering
   - Result formatting
   - Threshold filtering

================================================================================
INDEXING PIPELINE
================================================================================

Full pipeline flow:

1. Load Assessments (from catalog.json)
   ↓
2. Build Documents (1 per assessment with rich content)
   ↓
3. Chunk Documents (3 chunks per document: overview, skills, full)
   ↓
4. Generate Embeddings (with caching, batch processing)
   ↓
5. Store in ChromaDB (with full metadata)

Result: Semantic knowledge base ready for retrieval

================================================================================
CLI TOOLS
================================================================================

1. build_knowledge_base.py
   - Builds complete knowledge base from catalog
   - Usage: python scripts/build_knowledge_base.py [--rebuild]
   - Flags:
     --rebuild: Delete existing index and rebuild

2. validate_knowledge_base.py
   - Validates index integrity and quality
   - Performs test queries
   - Reports statistics

3. query_knowledge_base.py
   - Interactive query interface
   - Usage: python scripts/query_knowledge_base.py [query]
   - Tests semantic search

4. kb_stats.py
   - Display comprehensive statistics
   - Collection size, metadata coverage, storage

5. test_kb_demo.py
   - Quick demo without full indexing
   - Tests components individually

================================================================================
USAGE INSTRUCTIONS
================================================================================

STEP 1: Build Knowledge Base (inside Docker container)

docker exec conversational-shl-assessment-recommender-api-1 \
  python3 scripts/build_knowledge_base.py

This will:
- Load catalog.json (5 demo assessments)
- Build 5 documents
- Create ~15 chunks (3 per assessment)
- Generate embeddings (384 dimensions)
- Store in ChromaDB

Expected output:
- ~15 vectors indexed
- ~1-2 minutes on first run (model download)
- ~10-30 seconds on subsequent runs (cached)

STEP 2: Validate Index

docker exec conversational-shl-assessment-recommender-api-1 \
  python3 scripts/validate_knowledge_base.py

This will:
- Check collection exists
- Verify vector count
- Test retrieval
- Report statistics

STEP 3: Test Queries

docker exec conversational-shl-assessment-recommender-api-1 \
  python3 scripts/query_knowledge_base.py "cognitive reasoning"

STEP 4: View Statistics

docker exec conversational-shl-assessment-recommender-api-1 \
  python3 scripts/kb_stats.py
